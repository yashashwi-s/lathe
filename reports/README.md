# Reports

This directory contains the research narrative and rendered methodology for the
canonical 157-document benchmark. Start with these two artifacts:

| Artifact | Purpose |
|---|---|
| `benchmark_paper_blog.md` | Generated, readable account of the task, frozen primary protocol, scorecard, uncertainty, failures, exploratory analyses, validation evidence, limitations, and reproduction |
| `../output/pdf/pdf_fidelity_metric_system_v3.pdf` | Final visual methodology report with labeled examples and the model/engine scorecards |
| `pdf_fidelity_metric_system_v3.typ` | Rebuildable Typst source for the final PDF |

## Publication contract

`benchmark_paper_blog.md` is generated by:

```bash
mamba run -n lathe python scripts/evaluation/analyze_publication_results.py
mamba run -n lathe python scripts/evaluation/build_publication_manuscript.py
```

Its only numeric inputs are the JSON and CSV files in
`results/publication_v0/`. The builder validates the required artifact set and
schema before replacing the report, so a missing scorecard, incomplete row set,
or unsupported schema fails closed. Do not hand-edit reported values in the
generated file; fix or regenerate the publication artifact instead.

The generated manuscript keeps these evidence classes distinct:

- the frozen prompt-v1 held-out run is the primary AI protocol;
- deterministic-engine results are identified by their own assigned corpus;
- paired uncertainty is shown only when the publication artifact provides it;
- the adaptive prompt cascade is exploratory;
- the seven-case Claude overlap is a heterogeneous-protocol case study, not a
  model ranking;
- metric axes report eligibility and abstention, never a universal scalar.

No current claim depends on human ratings.
